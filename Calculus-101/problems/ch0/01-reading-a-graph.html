0.1 Reading a Function from its Graph0.1 由圖形判讀函數

1 · Concept and Fundamental Knowledge1 · 概念與基礎知識

Definition · Function定義 · 函數

A function is a rule that takes an input and returns exactly one output. We usually call the input \(x\) and the output \(y\), and we write \(y=f(x)\), read as “\(y\) equals \(f\) of \(x\).” The single most important word in that sentence is exactly one: for each allowed input there is one and only one output. A machine that sometimes gave two different answers for the same input would not be a function.函數是一條規則，吃進一個輸入，回傳恰好一個輸出。我們通常把輸入記為 \(x\)、輸出記為 \(y\)，並寫成 \(y=f(x)\)，讀作「\(y\) 等於 \(f\) of \(x\)」。這句話中最關鍵的字是恰好一個：每個允許的輸入只對應唯一一個輸出。若同一個輸入有時給出兩個不同答案，那就不是函數。

The graph of \(f\) is the picture you get by plotting every point \(\bigl(x,\,f(x)\bigr)\): the horizontal coordinate is the input, the vertical coordinate is the output. Reading a graph well is the first skill of the whole course, because limits, derivatives, and integrals are all described through graphs. Four things can be read directly off a graph:\(f\) 的圖形是把所有點 \(\bigl(x,\,f(x)\bigr)\) 畫出來所得到的圖：水平座標是輸入，垂直座標是輸出。能熟練判讀圖形是整門課的第一個技能，因為極限、導數與積分都透過圖形來描述。可由圖形直接讀出四件事：

Key terms · what you read off a graph重要名詞 · 由圖形可讀出

  • The value \(f(a)\). To find the output at an input \(a\), start at \(x=a\) on the horizontal axis, move straight up or down until you meet the curve, then read the height of that point on the vertical axis. That height is \(f(a)\).函數值 \(f(a)\)。要求輸入 \(a\) 處的輸出，先在水平軸找到 \(x=a\)，再垂直上下移動直到碰到曲線，然後讀出該點在垂直軸上的高度。這個高度就是 \(f(a)\)。
  • The domain. The domain is the set of all inputs the function accepts. On the graph it is the shadow of the curve cast onto the horizontal axis: the stretch of \(x\)-values that lie under (or over) some point of the curve.定義域。定義域是函數所接受的所有輸入所成的集合。在圖形上，它是曲線投影到水平軸上的影子：也就是曲線下方（或上方）所涵蓋的那段 \(x\) 值範圍。
  • The range. The range is the set of all outputs the function actually produces. On the graph it is the shadow of the curve cast onto the vertical axis: from the lowest height the curve reaches up to the highest.值域。值域是函數實際產生的所有輸出所成的集合。在圖形上，它是曲線投影到垂直軸上的影子：從曲線到達的最低高度到最高高度。
  • Endpoints: filled vs. open. A filled dot ● means that endpoint is included; an open dot ○ means it is excluded. We record this with interval notation: \([a,b]\) includes both ends, \((a,b)\) excludes both, and \([a,b)\) includes the left end only.端點：實心與空心。實心點 ● 表示該端點包含在內；空心點 ○ 表示不包含。我們用區間記號記錄：\([a,b]\) 含兩端、\((a,b)\) 兩端皆不含、\([a,b)\) 僅含左端。

Rule · The vertical line test規則 · 垂直線檢驗法

Because a function gives exactly one output per input, every vertical line may cross the graph at most once. If some vertical line hits a curve twice, that input would have two outputs, so the curve is not the graph of a function. Use this test whenever you are unsure whether a picture even represents a function.因為函數對每個輸入恰好給一個輸出，所以任一條垂直線與圖形最多只能相交一次。若某條垂直線與曲線相交兩次，該輸入就會有兩個輸出，故此曲線不是函數的圖形。當你不確定一張圖是否代表函數時，就用這個檢驗法。

2 · Worked Examples2 · 範例（詳解）

Example 1 · Both endpoints filled範例 1 · 兩端皆為實心

The graph below shows a function \(g\). It is drawn with two straight segments meeting at a peak, and both endpoints are filled dots. Read its domain, its range, and the value \(g(3)\).下圖是函數 \(g\)，由兩段相接於頂點的直線段構成，兩端皆為實心點。請讀出它的定義域、值域，以及函數值 \(g(3)\)。

Reading it off, step by step逐步判讀

Quantity量 Value值 How you read it判讀方式
Domain定義域 \([0,4]\) The curve’s shadow on the \(x\)-axis runs from the left filled end \(x=0\) to the right filled end \(x=4\); both are included.曲線在 \(x\) 軸的影子由左側實心端 \(x=0\) 延伸到右側實心端 \(x=4\)，兩端皆含。
Range值域 \([2,6]\) The lowest the curve goes is \(y=2\) (left end), the highest is \(y=6\) (the peak), so every height between 2 and 6 is hit.曲線最低到 \(y=2\)（左端），最高到 \(y=6\)（頂點），故 2 到 6 之間每個高度都被取到。
\(g(3)\) \(5\) Go up from \(x=3\) (dashed) to the curve, then across to the \(y\)-axis: the height is 5.由 \(x=3\) 沿虛線上移到曲線，再水平移到 \(y\) 軸：高度為 5。

Notice that \(g(3)=5\) even though there is no grid line at height 5: the point sits exactly halfway between the grid lines \(y=4\) and \(y=6\). Reading values often means interpolating between grid lines like this.注意即使高度 5 沒有格線，\(g(3)=5\) 仍成立：該點恰好落在格線 \(y=4\) 與 \(y=6\) 的正中間。判讀數值常常需要像這樣在格線之間做內插。

Example 2 · An open endpoint範例 2 · 一個空心端點

The next graph is a function \(p\) drawn as a single straight rising line. The left end is an open dot ○ (excluded) and the right end is a filled dot ● (included). Read its domain, its range, and the value \(p(2)\). Watch how one open endpoint changes a square bracket into a round one.下一張圖是函數 \(p\)，畫成一條上升的直線。左端是空心點 ○（不含），右端是實心點 ●（含）。請讀出它的定義域、值域，以及函數值 \(p(2)\)。注意一個空心端點如何把中括號變成小括號。

Reading it off, step by step逐步判讀

Quantity量 Value值 How you read it判讀方式
Domain定義域 \((0,4]\) The shadow on the \(x\)-axis runs from \(x=0\) to \(x=4\), but the left end is open, so \(x=0\) is excluded: a round bracket on the left, a square bracket on the right.在 \(x\) 軸的影子由 \(x=0\) 到 \(x=4\)，但左端為空心，故 \(x=0\) 不含：左用小括號、右用中括號。
Range值域 \((1,5]\) As \(x\) leaves the open left end the height drops toward \(y=1\) but never reaches it; the top height \(y=5\) is the filled right end, so it is included.當 \(x\) 離開左側空心端，高度趨近 \(y=1\) 卻取不到；最高高度 \(y=5\) 為實心右端，故包含在內。
\(p(2)\) \(3\) Go up from \(x=2\) (dashed) to the line, then across: the height is 3, halfway between \(y=2\) and \(y=4\).由 \(x=2\) 沿虛線上移到直線，再水平移過去：高度為 3，落在 \(y=2\) 與 \(y=4\) 的中間。

Rule of thumb · brackets follow the dot口訣 · 括號跟著點走

A filled endpoint keeps its value (square bracket); an open endpoint loses it (round bracket). The same open dot that excludes \(x=0\) from the domain also excludes \(y=1\) from the range.實心端點保留其值（中括號）；空心端點則失去（小括號）。同一個空心點既把 \(x=0\) 排除於定義域，也把 \(y=1\) 排除於值域。

Example 3 · A flat segment範例 3 · 一段水平線

This function \(q\) rises and then runs flat: it climbs from \((0,2)\) to a corner at \((2,6)\), then stays level out to \((4,6)\). All three marked points are filled. Read its domain, its range, and the values \(q(1)\) and \(q(3)\). A flat piece is a good reminder that many different inputs may share the same output.此函數 \(q\) 先上升、再走水平：由 \((0,2)\) 爬升到轉角 \((2,6)\)，之後維持水平到 \((4,6)\)。三個標示點皆為實心。請讀出它的定義域、值域，以及函數值 \(q(1)\) 與 \(q(3)\)。水平段正好提醒我們：許多不同的輸入可以共用同一個輸出。

Reading it off, step by step逐步判讀

Quantity量 Value值 How you read it判讀方式
Domain定義域 \([0,4]\) The shadow on the \(x\)-axis runs from the filled left end \(x=0\) to the filled right end \(x=4\); both included.在 \(x\) 軸的影子由實心左端 \(x=0\) 到實心右端 \(x=4\)，兩端皆含。
Range值域 \([2,6]\) The lowest height is \(y=2\) (left end); the highest is \(y=6\), held all along the flat part. Every height from 2 up to 6 is reached on the rise.最低高度為 \(y=2\)（左端）；最高為 \(y=6\)，並在整段水平處維持。上升途中 2 到 6 之間每個高度都被取到。
\(q(1)\) \(4\) \(x=1\) is halfway up the rising segment from \((0,2)\) to \((2,6)\), so the height is halfway between 2 and 6.\(x=1\) 位於由 \((0,2)\) 上升到 \((2,6)\) 那段的中點，故高度為 2 與 6 的中間值。
\(q(3)\) \(6\) \(x=3\) lands on the flat part, where the height is a constant \(6\).\(x=3\) 落在水平段上，高度固定為 \(6\)。

Notice · many inputs, one output is allowed注意 · 多個輸入對一個輸出是可以的

Both \(x=2\) and \(x=3\) (and every \(x\) between) give the output \(6\). That is perfectly fine: a function may send many inputs to one output. What it may not do is send one input to two outputs, which is exactly what the vertical line test forbids, and this graph passes it.\(x=2\) 與 \(x=3\)（以及其間每個 \(x\)）都給出輸出 \(6\)。這完全沒問題：函數可以把許多輸入送到同一個輸出。它不能做的是把一個輸入送到兩個輸出，而這正是垂直線檢驗法所禁止的，本圖通過該檢驗。

3 · Exercises3 · 練習

Exercise 1 · Both endpoints filled練習 1 · 兩端皆為實心

Now it is your turn. The graph below shows a function \(h\), again made of two straight segments, this time dipping to a low point. Both endpoints are filled dots.換你試試。下圖是函數 \(h\)，同樣由兩段直線構成，這次往下凹到一個低點。兩端皆為實心點。

Using the graph of \(h\), answer all five parts:利用 \(h\) 的圖形，回答以下五個小題：

  1. State the domain of \(h\) in interval notation.以區間記號寫出 \(h\) 的定義域。
  2. State the range of \(h\) in interval notation.以區間記號寫出 \(h\) 的值域。
  3. Find \(h(1)\).求 \(h(1)\)。
  4. Find \(h(3)\).求 \(h(3)\)。
  5. At which input \(x\) is the output smallest, and what is that smallest output?在哪個輸入 \(x\) 處輸出最小？該最小輸出值為何？
Hint:提示： For the domain and range, look at how far the curve spreads left-to-right and bottom-to-top. For \(h(1)\) and \(h(3)\), go up from that \(x\)-value to the curve and read the height; each lands neatly relative to the grid lines \(y=2,4,6\).求定義域與值域時，觀察曲線左右與上下延伸的範圍。求 \(h(1)\) 與 \(h(3)\) 時，由該 \(x\) 值往上到曲線再讀高度；兩者相對於格線 \(y=2,4,6\) 都落在好讀的位置。
Show full solution顯示完整解答

(1) Domain. The curve’s left filled endpoint sits at \(x=0\) and its right filled endpoint at \(x=4\); the function is drawn for every \(x\) in between. Because both endpoints are filled, both are included:(1) 定義域。曲線左側實心端在 \(x=0\)，右側實心端在 \(x=4\)；中間每個 \(x\) 皆有定義。由於兩端皆為實心，兩端皆包含：

\[ \text{domain}(h)=[0,4]. \]

(2) Range. Scan the curve from its lowest point to its highest. The lowest it reaches is the dip at height \(y=1\); the highest is the left endpoint at height \(y=7\). Every height between is achieved on the way down and up, so:(2) 值域。從曲線最低處掃到最高處。最低落在凹點高度 \(y=1\)；最高在左端點高度 \(y=7\)。下降與上升途中之間每個高度都被取到，故：

\[ \text{range}(h)=[1,7]. \]

(3) \(h(1)\). Start at \(x=1\) and move up to the curve. The point \(x=1\) is halfway along the falling segment from \((0,7)\) to \((2,1)\), so its height is halfway between 7 and 1, which lands exactly on the grid line \(y=4\):(3) \(h(1)\)。由 \(x=1\) 往上到曲線。\(x=1\) 位於由 \((0,7)\) 下降到 \((2,1)\) 那段的中點，故高度為 7 與 1 的中間值，恰好落在格線 \(y=4\) 上：

\[ h(1)=4. \]

(4) \(h(3)\). Now \(x=3\) is halfway along the rising segment from \((2,1)\) to \((4,5)\). Its height is halfway between 1 and 5:(4) \(h(3)\)。此時 \(x=3\) 位於由 \((2,1)\) 上升到 \((4,5)\) 那段的中點。其高度為 1 與 5 的中間值：

\[ h(3)=\frac{1+5}{2}=3. \]

(5) Smallest output. The curve falls and then rises, so its lowest point is the corner where the two segments meet. That happens at \(x=2\), where the height is \(1\):(5) 最小輸出。曲線先降後升，故最低點是兩段相接的轉角處，發生在 \(x=2\)，高度為 \(1\)：

\[ \text{smallest output } h(2)=1 \text{ at } x=2. \]

Sense check. The smallest output \(1\) is exactly the bottom of the range \([1,7]\), and the largest output \(7\) is the top, taken at the left endpoint \(x=0\). The two readings \(h(1)=4\) and \(h(3)=3\) both lie inside \([1,7]\), as every output must. Everything is consistent.合理性檢查。最小輸出 \(1\) 恰為值域 \([1,7]\) 的下界，最大輸出 \(7\) 為上界，取於左端點 \(x=0\)。兩個讀值 \(h(1)=4\) 與 \(h(3)=3\) 都落在 \([1,7]\) 內，符合每個輸出都應落在值域中的要求。一切一致。

Exercise 2 · An open endpoint練習 2 · 一個空心端點

Here is a function \(m\) made of two straight segments that dip and rise back up. The left end \(\bigl(0,6\bigr)\) is filled ●, but the right end \(\bigl(4,6\bigr)\) is an open dot ○. Read carefully: this open dot is the whole point of the exercise.這是函數 \(m\)，由兩段先降後升的直線構成。左端 \(\bigl(0,6\bigr)\) 為實心 ●，但右端 \(\bigl(4,6\bigr)\) 為空心點 ○。請仔細判讀：這個空心點正是本練習的重點。

Using the graph of \(m\), answer all five parts:利用 \(m\) 的圖形，回答以下五個小題：

  1. State the domain of \(m\) in interval notation. (Mind the open right end.)以區間記號寫出 \(m\) 的定義域。（注意右側空心端。）
  2. State the range of \(m\) in interval notation.以區間記號寫出 \(m\) 的值域。
  3. Find \(m(1)\).求 \(m(1)\)。
  4. Find \(m(3)\).求 \(m(3)\)。
  5. The height \(6\) is reached at the left end. Is it also reached at \(x=4\)? Explain using the open dot.高度 \(6\) 在左端達到。在 \(x=4\) 處是否也達到？請用空心點說明。
Hint:提示： The open dot at the right end means \(x=4\) is not in the domain, so the bracket on that side is round. For the range, the top height \(6\) is still included because the left end at \((0,6)\) is filled.右端的空心點表示 \(x=4\) 不在定義域內，故該側用小括號。至於值域，最高高度 \(6\) 仍被包含，因為左端 \((0,6)\) 是實心。
Show full solution顯示完整解答

(1) Domain. The shadow on the \(x\)-axis runs from \(x=0\) to \(x=4\). The left end is filled (included) but the right end is open (excluded), so the right bracket is round:(1) 定義域。在 \(x\) 軸的影子由 \(x=0\) 到 \(x=4\)。左端實心（含）、右端空心（不含），故右側用小括號：

\[ \text{domain}(m)=[0,4). \]

(2) Range. The lowest height is the dip at \(y=2\) (a filled corner, included). The highest height is \(y=6\). Even though the right end \((4,6)\) is open, the left end \((0,6)\) is filled, so \(y=6\) is still produced and the top bracket is square:(2) 值域。最低高度為凹點 \(y=2\)（實心轉角，含）。最高高度為 \(y=6\)。雖然右端 \((4,6)\) 為空心，但左端 \((0,6)\) 為實心，故 \(y=6\) 仍被取到，上界用中括號：

\[ \text{range}(m)=[2,6]. \]

(3) \(m(1)\). \(x=1\) is halfway down the falling segment from \((0,6)\) to \((2,2)\), so the height is halfway between 6 and 2:(3) \(m(1)\)。\(x=1\) 位於由 \((0,6)\) 下降到 \((2,2)\) 那段的中點，故高度為 6 與 2 的中間值：

\[ m(1)=\frac{6+2}{2}=4. \]

(4) \(m(3)\). \(x=3\) is halfway up the rising segment from \((2,2)\) to \((4,6)\). The endpoint \((4,6)\) being open does not affect \(x=3\), which sits strictly inside the segment:(4) \(m(3)\)。\(x=3\) 位於由 \((2,2)\) 上升到 \((4,6)\) 那段的中點。端點 \((4,6)\) 為空心並不影響 \(x=3\)，因為它嚴格落在線段內部：

\[ m(3)=\frac{2+6}{2}=4. \]

(5) Is \(6\) reached at \(x=4\)? No. The dot at \((4,6)\) is open, so \(x=4\) is not in the domain and \(m(4)\) is undefined. The height \(6\) is reached only at the left end \(x=0\). The curve climbs toward \((4,6)\) and gets as close as you like, but never arrives there.(5) \(6\) 在 \(x=4\) 處是否達到？否。\((4,6)\) 處的點為空心，故 \(x=4\) 不在定義域內，\(m(4)\) 無定義。高度 \(6\) 僅在左端 \(x=0\) 達到。曲線朝 \((4,6)\) 攀升、可任意接近，卻永遠到不了該點。

Sense check. Both readings \(m(1)=4\) and \(m(3)=4\) lie inside the range \([2,6]\); the two segments are mirror images, so equal heights at \(x=1\) and \(x=3\) make sense. The smallest output \(2\) (at \(x=2\)) is the bottom of the range, exactly as it should be.合理性檢查。兩個讀值 \(m(1)=4\) 與 \(m(3)=4\) 都落在值域 \([2,6]\) 內；兩段互為鏡像，故 \(x=1\) 與 \(x=3\) 高度相等合理。最小輸出 \(2\)（在 \(x=2\)）為值域下界，恰如預期。
