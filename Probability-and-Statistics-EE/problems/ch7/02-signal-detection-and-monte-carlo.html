7.2 Signal Detection and Monte Carlo7.2 訊號偵測與蒙地卡羅

1 · Concept and Fundamental Knowledge1 · 概念與基礎知識

Why an electronic engineer needs this. A receiver often decides whether a 0 or 1 was sent after noise has shifted the voltage. Signal detection turns that decision into a threshold problem, and Monte Carlo simulation estimates the error rate when formulas are hard.電子工程師為什麼需要這個。接收器常在雜訊改變電壓後判斷送出的是 0 還是 1。訊號偵測把這個判斷轉成門檻問題，而蒙地卡羅模擬可在公式困難時估計錯誤率。

Definition · threshold detector定義 · 門檻偵測器

A threshold detector compares a received value \(Y\) with a threshold \(c\). If \(Y\gt c\), decide 1; otherwise decide 0. An error happens when noise pushes \(Y\) to the wrong side.門檻偵測器把接收值 \(Y\) 與門檻 \(c\) 比較。若 \(Y\gt c\) 則判定為 1，否則判定為 0。當雜訊把 \(Y\) 推到錯邊時，就發生錯誤。

Simulate it first先模擬看看

Predict: if 0 and 1 are equally likely, noise has standard deviation 0.25, and the threshold is 0.5, what error rate do you expect?預測：若 0 與 1 等可能，雜訊標準差為 0.25，門檻為 0.5，你預期錯誤率約多少？

import numpy as np
rng = np.random.default_rng(11)

bits = rng.integers(0, 2, size=300_000)
noise = rng.normal(0, 0.25, size=bits.size)
received = bits + noise
decisions = received > 0.5
print((decisions != bits).mean())

Observed: about \(0.023\). Errors are rare because the threshold is two noise standard deviations from each signal level.觀察結果：約 \(0.023\)。錯誤不多，因為門檻離每個訊號電平約兩個雜訊標準差。

Key terms · detection errors重要名詞 · 偵測錯誤

  • False alarm. Decide 1 when 0 was sent.誤警。送出 0 卻判定為 1。
  • Miss. Decide 0 when 1 was sent.漏偵。送出 1 卻判定為 0。
  • Monte Carlo estimate. Repeat random trials and use the sample fraction as an estimated probability.蒙地卡羅估計。重複隨機試驗，以樣本比例估計機率。

Rule · error rate from conditional errors規則 · 由條件錯誤率求總錯誤率

\[ P(\text{error})=P(0)P(Y\gt c\mid 0)+P(1)P(Y\le c\mid 1). \]

For equally likely bits, average the false-alarm probability and the miss probability.若位元等可能，總錯誤率就是誤警機率與漏偵機率的平均。

2 · Worked Examples2 · 範例（詳解）

Example 1 · A symmetric threshold範例 1 · 對稱門檻

The signal levels are 0 and 1, noise standard deviation is 0.20, and the threshold is 0.5.訊號電平為 0 與 1，雜訊標準差為 0.20，門檻為 0.5。

Reading it off, step by step逐步判讀

Quantity量Value值How you read it判讀方式
\(c\)\(0.5\)Middle of the two signal levels.兩個訊號電平的中點。
\(z\)\(0.5/0.20=2.5\)Threshold is 2.5 standard deviations from either mean.門檻離任一平均值 2.5 個標準差。
\(P_e\)\(\approx 0.0062\)Each tail probability is about 0.0062.每側尾端機率約為 0.0062。

The plain case is symmetric, so false alarm and miss have the same probability.基本情況是對稱的，因此誤警與漏偵機率相同。

Example 2 · A shifted threshold範例 2 · 偏移門檻

Keep the same levels and noise, but move the threshold to \(c=0.6\). Compare the two error types.維持相同電平與雜訊，但把門檻移到 \(c=0.6\)。比較兩種錯誤。

Reading it off, step by step逐步判讀

Quantity量Value值How you read it判讀方式
False alarm誤警\(P(Y\gt 0.6\mid 0)\)Right tail of the 0 distribution.0 分布的右尾。
Miss漏偵\(P(Y\le 0.6\mid 1)\)Left tail of the 1 distribution.1 分布的左尾。
Effect效果fewer false alarms較少誤警The detector becomes more conservative about deciding 1.偵測器對判定 1 變得較保守。

The wrinkle is a tradeoff: changing the threshold helps one error type and hurts the other.這題的重點是取捨：改變門檻會改善一種錯誤，卻惡化另一種。

Example 3 · Receiver error by simulation (engineering)範例 3 · 以模擬估計接收器錯誤（工程應用）

A receiver uses \(c=0.5\), levels 0 and 1, and \(\sigma=0.35\). Estimate the bit error rate with Monte Carlo.某接收器使用 \(c=0.5\)、電平 0 與 1、\(\sigma=0.35\)。用蒙地卡羅估計位元錯誤率。

Reading it off, step by step逐步判讀

Quantity量Value值How you read it判讀方式
Trials試驗many bits多個位元Randomly generate sent bits and noise.隨機產生傳送位元與雜訊。
Decision判定\(Y\gt 0.5\)Apply the same detector to every trial.對每次試驗套用同一偵測器。
Estimate估計\(\text{errors}/\text{trials}\)The sample error fraction estimates BER.樣本錯誤比例估計 BER。

Engineering simulation is valuable because the same code can handle unequal priors, clipping, or non-Gaussian noise.工程模擬很有價值，因為同一段程式可處理不等先驗、截波或非高斯雜訊。

3 · Exercises3 · 練習

Exercise 1 · Label the two errors練習 1 · 標示兩種錯誤

A detector uses threshold \(c=0.4\) for levels 0 and 1.某偵測器對電平 0 與 1 使用門檻 \(c=0.4\)。

  1. Write the false-alarm event.寫出誤警事件。
  2. Write the miss event.寫出漏偵事件。
  3. Which error becomes larger when the threshold moves down?門檻下移時，哪一種錯誤會變大？
Hint:提示： A lower threshold makes it easier to decide 1.較低門檻會讓判定為 1 更容易。
Show full solution顯示完整解答

False alarm means 0 was sent but the received value crosses the threshold:誤警表示送出 0，但接收值跨過門檻：

\[ P(Y\gt 0.4\mid 0). \]

Miss means 1 was sent but the received value stays below the threshold:漏偵表示送出 1，但接收值低於門檻：

\[ P(Y\le 0.4\mid 1). \]

Moving the threshold down increases false alarms and decreases misses.門檻下移會增加誤警並減少漏偵。

Sense check. A very low threshold would almost always decide 1, so false alarms should dominate.合理性檢查。非常低的門檻幾乎總是判定 1，因此誤警應占主導。

Exercise 2 · Estimate by Monte Carlo練習 2 · 用蒙地卡羅估計

Use levels 0 and 1, threshold 0.5, and noise standard deviation 0.30.使用電平 0 與 1、門檻 0.5、雜訊標準差 0.30。

  1. Write the simulation steps.寫出模擬步驟。
  2. Estimate the error rate with many trials.用多次試驗估計錯誤率。
  3. Explain why increasing the number of trials helps.說明為何增加試驗次數有幫助。
Hint:提示： Generate bits, add normal noise, apply the threshold, then count mismatches.產生位元、加入常態雜訊、套用門檻，再計算不一致的比例。
Show full solution顯示完整解答

The Monte Carlo estimate is the fraction of decisions different from the sent bits.蒙地卡羅估計值就是判定結果與送出位元不同的比例。

\[ \widehat P_e=\frac{\text{number of errors}}{\text{number of trials}}. \]

For this setup, a large simulation gives about \(0.048\).在此設定下，大量模擬約得到 \(0.048\)。

More trials reduce random simulation noise, so repeated runs move closer together.更多試驗會降低模擬本身的隨機波動，因此重複執行的結果會更接近。

Sense check. The error rate is larger than the \(\sigma=0.25\) case because the distributions overlap more.合理性檢查。此錯誤率比 \(\sigma=0.25\) 時大，因為兩個分布重疊更多。
