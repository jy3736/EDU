1. 教具定位

FormativeX 是由南臺科技大學電子工程系楊榮林教授自行設計與實作的課堂形成性評量與教學分析平台，實際應用於 JavaScript 程式設計（五專電子三甲，30D1GH01）課程。作品完整名稱可表述為「AI 協作學習評量與教學分析平台：智慧題庫生成、即時監測、學習診斷與教學決策輔助」。

本作品以 Flask 架構開發，整合學生端線上作答、教師端即時儀表板、題庫管理、課後分析與報告匯出。它要改善的不是單一考試流程，而是傳統紙本小考、通用表單測驗與人工批改所造成的流程繁瑣、資料分散、回饋延遲與難以長期追蹤等問題。

模組檔案使用者主要用途
學生作答端exam.py學生登入、選擇測驗、限時作答、提交、查看成績、下載個人作答報告
教師儀表板dashboard.py教師題組啟用、測驗伺服器控制、即時監測、錯題分析、成績分布、詳解匯出、成績下載

這套系統並不是將現有商業測驗平台簡單套用到課堂，而是針對實際教學現場長期觀察到的問題，利用課餘與下班後時間逐步設計、實作、修正而成的獨立教具。其開發目的，是讓教師能掌握題目生成、測驗環境、資料保存、學習分析與課後回饋的完整流程，而不必受限於外部平台的功能邊界、網路條件或資料格式。

2. 施測

5. 回饋
學生下載報告，教師調整教學。

2. 開發動機與設計理念

FormativeX 的誕生，來自幾個非常具體的教學需求。第一，生成式 AI 普及後，傳統線上測驗若完全依賴外部網站與網際網路，測驗公平性會快速下降。學生在測驗期間只要能連上外部服務，就可能使用 AI 工具查詢答案；但若完全不用數位工具，教師又會失去即時批改、錯題分析與長期追蹤的能力。因此，本系統選擇以「教室區域網路內可獨立運作」作為核心設計，使教師可以在必要時切斷對外網路，同時仍保留數位評量與即時分析。

第二，既有工具如 Zuvio、Firebase 雲端服務、GitHub Pages 或其他外部平台，雖然能處理部分表單、測驗或資料儲存需求，但很難完全符合程式設計課堂中的細緻需求。例如：多於四個選項的題型、依學生與試卷固定亂序、測驗中即時查看每題正確狀態、記錄每題停留時間、即時辨識亂猜行為、匯出個人化作答卷、同時整合多次形成性評量等。FormativeX 因此選擇完全自行實作，讓平台功能能直接回應教學現場，而不是讓教學流程遷就外部工具。

第三，這套教具強調「資料留在教師可掌握的環境中」。學生名單、作答紀錄、成績、IP、作答歷程與刪除紀錄都保存在本機或課程資料夾中，不需要上傳至第三方雲端服務。這對含有個資與學習歷程資料的課堂評量特別重要，也讓教師能依自己的課程節奏備份、整理與分析。

第四，開發此平台的目的不是減少教師參與，而是讓教師能更深入參與教學判斷。AI 可以協助生成題目，但題目是否符合教材、難度是否合理、選項是否具有辨識度、錯題是否代表共同迷思，仍由教師決定。系統負責收集與呈現資料，真正的教學決策仍回到教師身上。

FormativeX 可被視為一套「教師主導、AI 輔助、區域網路可獨立部署」的教學分析工具。它反映的是教師為了改善課堂出席、提升形成性評量品質、降低猜題干擾、限制測驗期間 AI 存取、並讓每次小測驗都能回到教學循環中，而投入大量非正式工作時間逐步完成的教具系統。

3. 教學特色

多選項題型

支援 A-Z 任意選項數，可製作四選一、五選一、六選一或七選一題目，降低純猜測取得高分的機率。

封閉式區域網路

平台可在教室 LAN 運作，不依賴外部商業測驗平台；正式施測可中斷外網以限制測驗期間的 AI 存取，同時保留數位評量與即時分析能力。

即時學習歷程

記錄題目停留時間、答案修改、回看題目與作答進度曲線，讓教師掌握過程而不只看分數。

測驗轉化為教材

測驗結束後即可匯出統計、詳解與學生個人化報告，成為課後複習與補救教學資料。

獨立部署特性

FormativeX 的執行不需要 Zuvio、Firebase、GitHub、雲端資料庫或第三方測驗平台。只要教室內有一台可執行 Python/Flask 的電腦作為伺服器，學生與教師端瀏覽器能連到同一個區域網路位址，即可完成測驗流程。

這種設計特別適合生成式 AI 時代的教室測驗：教師可保留數位系統的即時分析能力，同時透過切斷外網限制學生在測驗中使用 ChatGPT、搜尋引擎或其他線上資源。

6. 教師端使用手冊

啟動教師儀表板

cd formativeX-製作教具
COURSE_ID="py1b2026spring" python3 dashboard.py

預設教師端位址為 http://<server-ip>:5050。教師可從儀表板選擇班級、設定題組、啟動或停止學生作答服務。

啟用題組

  1. 進入 Dashboard 主畫面。
  2. 使用 New、All、Picked Up、Completed 等狀態篩選。
  3. 使用單元編號篩選需要的題組。
  4. 勾選本次要開放的測驗題組。
  5. 儲存設定後，學生端只會顯示已啟用題組。

啟動學生作答服務

EXAM_SIGNATURE_KEY="your-strong-random-secret" \
COURSE_ID="py1b2026spring" \
python3 exam.py --time 20

即時監測與管理

「測驗狀態」分頁以學生卡片呈現未開始、作答中與已繳交狀態；點選卡片可查看個別學生作答細節。「管理測驗」分頁可強制放棄、強制繳交，或設定最後 N 分鐘。

匯出詳解與成績

「題目詳解」可匯出單一或多題組 Markdown、HTML 或 ZIP。「成績下載」可選擇多份題組，設定完成題數權重與答對題數權重，並輸出 JSON、HTML 或 Excel。

成績計算：成績 = min(a × 完成率 + b × 答對率, 100)

7. 學生端使用手冊

登入

  1. 開啟教師提供的學生端網址，預設為 http://<server-ip>:5000。
  2. 輸入學號。
  3. 系統自動顯示姓名，確認身份。
  4. 選擇可作答測驗並進入。

作答

學生一次只看到一題；左側顯示測驗名稱、姓名、學號與剩餘時間。學生可切換上一題、下一題，也可勾選「作答後自動下一題」。

交卷與下載個人報告

提交後系統立即批改，顯示答對題數與表現評語。學生可下載完整作答卷，內容包含個人答案、正確答案、各題對錯與全班相對表現。

以下截圖說明不只描述畫面元件，也補充開發者設計這些功能時的教學意圖。FormativeX 的重點不是把紙本小考搬到網頁上，而是把題庫生成、封閉式區域網路施測、即時監測、錯題分析、個別化回饋與長期學習資料累積，整合成教師可自行掌握的教學工具。

9. 建議教學流程

課前

  1. 依教材與學習目標建立或更新題組。
  2. 使用 AI 工作流程產生候選題目。
  3. 教師人工審查題意、答案與難度。
  4. 將題庫放入 courses/<course_id>/assessments/。
  5. 在 Dashboard 啟用本次測驗題組。

課中

  1. 啟動學生作答服務，讓學生完成前測或後測。
  2. 教師即時查看未到、遲到或作答異常情況。
  3. 使用錯題分析與個別學生歷程進行補救教學。

課後

  1. 匯出題目詳解與測驗結果。
  2. 將報告上傳課程網站供學生複習。
  3. 分析長期完成率、答對率與穩定度，規劃下一次教學調整。

10. 維運與注意事項

  • submits/、active_logins/ 可能含有個資，應妥善保存。
  • 正式測驗建議使用固定 EXAM_SIGNATURE_KEY。
  • 若啟用 IP allowlist，需確認教室教師端與管理端 IP 在允許清單內。
  • 刪除作答紀錄時系統會搬移至 deleted/，但仍建議定期備份。
  • AI 生成題目必須經教師人工審核，尤其是程式輸出、非同步流程、選項唯一性與答案一致性。
  • 課堂施測不需 Zuvio、Firebase、GitHub 或其他外部雲端平台；這些工具即使可作為開發或備份輔助，也不是系統運作條件。
FormativeX 的價值不只在於自動批改，而在於把每一次小測驗轉化為可累積、可分析、可回饋的學習資料。它完全由教師自行開發與維護，能在教室區域網路中獨立運作，必要時可與外網隔離，以限制測驗期間的 AI 存取，同時保留數位評量最重要的即時回饋與長期分析能力。

11. 結語

FormativeX 將 AI 題組生成、區域網路測驗、即時監測、學習歷程分析、錯題診斷、成績下載與學生個人化回饋整合為一套完整教具。它的價值不只在於自動批改，而在於讓教師能在課堂當下掌握學生理解狀況，立即調整教學，並把每一次小測驗轉化為可累積、可分析、可回饋的學習資料。

這套系統也說明了一個明確的教學立場：面對 AI 時代，教師不應只是在既有工具中被動選擇，而可以依照自己的課程、學生與教室條件，建立真正符合教學現場的工具。FormativeX 完全由教師自行開發與維護，能在教室區域網路中獨立運作，必要時可與外網隔離，以限制測驗期間的 AI 存取，同時保留數位評量最重要的即時回饋與長期分析能力。
